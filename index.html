Modern event processing

Get Started Read the docs

Kaskada is a modern compute engine designed for processing events in bulk and real-time. Written in Rust and build on Apache Arrow, Kaskada can compute most workloads without the complexity and overhead of distributed execution.

Kaskada’s query language builds on the best features of SQL to provide a more expressive way to compute over events. Queries are declarative, simple, and composable. Unlike SQL they’re also concise and designed from the start for processing events chronologically, making it easier to reason about changes over time.

Compute from events as timelines

The foundation of Kaskada’s query language is the idea of a “timeline”.

Timelines describe a value that changes over time—in the same way that SQL queries transform a set of input tables into a result table, Kaskada queries transform a set of input timelines into a result timeline.

Thinking with timelines makes it easier to describe and reason about time and behavior.
